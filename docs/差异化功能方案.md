# 语音输入法 — 差异化功能方案

> 基于市场调研（闭源产品 + 开源生态 + 用户需求），提炼出 5 个可行的差异化方向。
> 目标：将项目从"语音输入工具"升级为"面向 AI 编程的语音输入工具"。

---

## 市场调研结论摘要

### 竞品全景

| 类别 | 代表产品 | 共同弱点 |
|------|---------|---------|
| **闭源商业** | Wispr Flow ($144/年)、SuperWhisper ($60/年) | 贵、macOS 独占、隐私顾虑 |
| **国内输入法** | 讯飞、搜狗 | 广告多、不懂编程术语、PC端体验差 |
| **系统自带** | Win+H、macOS Dictation | 中文差、无自定义词汇、不懂代码 |
| **开源项目** | Buzz (17K⭐)、WhisperWriter、Handy | 几乎全依赖本地GPU、无中文优化、UI简陋 |

### 核心市场空白

**Windows 平台上，面向 AI 编程场景的免费语音输入工具几乎不存在。**

### 本项目已有优势

1. Windows 原生 + 现代 UI（QFluentWidgets）
2. 云端 ASR 引擎（阿里云 Qwen3-ASR + Groq Whisper），不需要本地 GPU
3. 按住热键说话 + 自动粘贴的输入法工作流
4. 已有 AI 润色功能

## 功能一：AI Prompt 模式（语音→结构化 prompt）

### 原始需求

程序员用语音给 AI 下指令时，口语化表达（"那个用户列表改成分页的吧，每页20条，再加个搜索"）直接粘贴给 AI 效果不好。需要自动将口语润色为结构化 prompt。

**与现有功能的关系**：当前 `ai_polish` 开关只做"纠错+去口语词"，新功能是在此基础上增加一个独立的**润色模式选择**。

## 功能二：从项目代码自动提取词汇表

### 原始需求

程序员项目中有大量专业术语（库名、类名、函数名），手动维护 `custom_vocab` 很麻烦。希望能扫描一个项目目录，自动提取术语合并到词汇表。

### 设计

**新增模块级函数 `extract_vocab_from_project`**：

```python
def extract_vocab_from_project(project_dir: str) -> list[str]:
    """扫描项目目录，提取编程术语"""
    vocab = set()
    project = Path(project_dir)

    # 1. requirements.txt -> 包名
    req = project / "requirements.txt"
    if req.exists():
        for line in req.read_text(encoding="utf-8").splitlines():
            line = line.strip()
            if line and not line.startswith("#"):
                name = re.split(r"[>=<!\[\]]", line)[0].strip()
                if name:
                    vocab.add(name)

    # 2. package.json -> dependencies 键名
    pkg = project / "package.json"
    if pkg.exists():
        data = json.loads(pkg.read_text(encoding="utf-8"))
        for key in ("dependencies", "devDependencies"):
            if key in data:
                vocab.update(data[key].keys())

    # 3. pyproject.toml -> dependencies 列表
    pyproj = project / "pyproject.toml"
    if pyproj.exists():
        text = pyproj.read_text(encoding="utf-8")
        for m in re.finditer(r'"([A-Za-z][\w.-]*)(?:[>=<].*?)?"', text):
            vocab.add(m.group(1))

    # 4. Python import 语句（扫描 *.py / *.pyw）
    for ext in ("*.py", "*.pyw"):
        for f in project.rglob(ext):
            try:
                content = f.read_text(encoding="utf-8", errors="ignore")
                for m in re.finditer(
                    r"(?:from|import)\s+([\w.]+)", content
                ):
                    parts = m.group(1).split(".")
                    vocab.update(parts)
            except Exception:
                continue

    # 5. 过滤：去掉太短的（<=2字符）和 Python 内置模块
    builtins_skip = {
        "os", "sys", "re", "io", "json", "time", "math",
        "copy", "typing", "abc", "ast",
    }
    return sorted(
        v for v in vocab
        if len(v) > 2 and v not in builtins_skip
    )
```

## 功能三：识别结果预览确认

### 原始需求

当前识别完成后立即自动粘贴，如果识别有误会直接发给 AI，无法补救。需要一个预览浮窗，用户确认后再粘贴。

### 设计

**新增配置字段**：

```python

```

**新增 `PreviewPopup` 类**：

一个屏幕底部居中的半透明浮窗，显示识别文本，提供三个操作：

```

│  帮我把用户列表改成分页的每页二十条   │
```

```python
class PreviewPopup(QWidget):
    confirmed = Signal(str)   # 用户确认，携带最终文本
    cancelled = Signal()      # 用户取消

    def __init__(self):
        super().__init__(None)
        self.setWindowFlags(
            Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint | Qt.Tool
        )
        self.setAttribute(Qt.WA_TranslucentBackground)
        self.setFixedWidth(500)

        layout = QVBoxLayout(self)
        layout.setContentsMargins(16, 12, 16, 12)

        # 可编辑文本框（用户可以快速修改）
        self._text_edit = TextEdit(self)
        self._text_edit.setFixedHeight(80)
        layout.addWidget(self._text_edit)

        # 按钮行
        btn_row = QHBoxLayout()
        self._confirm_btn = PrimaryPushButton("粘贴 (Enter)", self)
        self._cancel_btn = PushButton("取消 (Esc)", self)
        btn_row.addStretch()
        btn_row.addWidget(self._confirm_btn)
        btn_row.addWidget(self._cancel_btn)
        layout.addLayout(btn_row)

        self._confirm_btn.clicked.connect(self._on_confirm)
        self._cancel_btn.clicked.connect(self._on_cancel)

    def show_text(self, text: str):
        self._text_edit.setPlainText(text)
        self.adjustSize()
        screen = QApplication.primaryScreen().geometry()
        x = (screen.width() - self.width()) // 2
        y = screen.height() - self.height() - 120
        self.move(x, y)
        self.show()
        self.raise_()
        self._text_edit.setFocus()

    def keyPressEvent(self, event):
        if event.key() == Qt.Key_Return and not event.modifiers():
            self._on_confirm()
        elif event.key() == Qt.Key_Escape:
            self._on_cancel()
        else:
            super().keyPressEvent(event)

    def _on_confirm(self):
        self.confirmed.emit(self._text_edit.toPlainText().strip())
        self.hide()

    def _on_cancel(self):
        self.cancelled.emit()
        self.hide()
```

## 功能四：连续录音模式

### 原始需求

写复杂 prompt 时需要思考停顿，当前"按住说话"模式不支持中途暂停。需要一个"短按切换录音/暂停，直到手动结束"的模式。

### 设计

**新增配置字段**：

```python
"recording_mode": "hold",  # "hold"=按住说话（现有）, "toggle"=连续模式
```

**连续模式的交互设计**：

```
短按热键 -> 开始录音（浮窗显示 "● 录音中"）
短按热键 -> 暂停录音（浮窗显示 "⏸ 已暂停，再按继续"）
短按热键 -> 恢复录音（浮窗显示 "● 录音中"）
双击热键 -> 结束录音，开始识别
```

**OverlayBubble 新增状态**：

```python
"paused": ("#6B7280", "white", "⏸  已暂停，再按继续"),
```

**AudioRecorder 新增暂停/恢复**：

```python
def pause(self) -> None:
    self.recording = False  # _callback 中不再收集数据

def resume(self) -> None:
    self.recording = True
```

不需要停止 stream，只需控制 `_callback` 中是否收集数据（`self.recording` 标志位已有此语义）。

**热键处理逻辑变更**：

```python
# VoiceInputApp 新增状态
self._toggle_state = "idle"  # "idle" | "recording" | "paused"
self._last_press_time = 0.0

def _on_hotkey_press(self, event):
    if self.config["recording_mode"] == "hold":
        # 原有逻辑不变
        ...
    else:
        self._on_toggle_press()

def _on_toggle_press(self):
    now = time.time()
    # 双击检测（300ms 内连按两次）
    if now - self._last_press_time < 0.3 and self._toggle_state != "idle":
        self._finish_toggle_recording()
        self._last_press_time = 0.0
        return
    self._last_press_time = now

    # 延迟 300ms 判断是否为单击
    threading.Timer(0.3, self._handle_single_toggle).start()

def _handle_single_toggle(self):
    if self._last_press_time == 0.0:
        return  # 已被双击处理
    if self._toggle_state == "idle":
        self._toggle_state = "recording"
        self.recorder.start()
        self.signals.overlay_show.emit("recording")
    elif self._toggle_state == "recording":
        self._toggle_state = "paused"
        self.recorder.pause()
        self.signals.overlay_show.emit("paused")
    elif self._toggle_state == "paused":
        self._toggle_state = "recording"
        self.recorder.resume()
        self.signals.overlay_show.emit("recording")

def _finish_toggle_recording(self):
    self._toggle_state = "idle"
    self.signals.overlay_show.emit("recognizing")
    threading.Thread(target=self._process_audio, daemon=True).start()
```

**hold 模式下 `_on_hotkey_release` 不变**，toggle 模式下 release 事件忽略。

```

```

## 功能五：应用感知自动切换润色模式

### 原始需求

不同场景需要不同的文本处理方式：在终端/IDE 中说话应该用 Prompt 模式，在浏览器聊天中应该用通用润色或不润色。希望自动识别当前窗口并切换。

### 设计

**新增配置字段**：

```python
"smart_mode": False,  # 是否启用应用感知
"smart_mode_rules": {
    "terminal": "prompt",    # 终端/命令行 -> Prompt 模式
    "ide": "prompt",         # IDE -> Prompt 模式
    "browser": "polish",     # 浏览器 -> 通用润色
    "default": "off",        # 其他应用 -> 不润色
}
```

**新增函数 `detect_foreground_app`**：

```python
def detect_foreground_app() -> str:
    """检测当前前台窗口，返回应用类别"""
    try:
        hwnd = ctypes.windll.user32.GetForegroundWindow()
        length = ctypes.windll.user32.GetWindowTextLengthW(hwnd)
        buf = ctypes.create_unicode_buffer(length + 1)
        ctypes.windll.user32.GetWindowTextW(hwnd, buf, length + 1)
        title = buf.value.lower()

        terminal_keywords = [
            "cmd", "powershell", "terminal", "bash", "mintty",
            "windows terminal", "claude", "warp",
        ]
        ide_keywords = [
            "visual studio code", "vscode", "cursor", "pycharm",
            "intellij", "webstorm", "sublime", "vim", "neovim",
        ]
        browser_keywords = [
            "chrome", "firefox", "edge", "safari", "brave", "opera",
        ]

        for kw in terminal_keywords:
            if kw in title:
                return "terminal"
        for kw in ide_keywords:
            if kw in title:
                return "ide"
        for kw in browser_keywords:
            if kw in title:
                return "browser"
        return "default"
    except Exception:
        return "default"
```

开启后润色模式 ComboBox 变灰（由智能模式接管），下方展开规则配置：

```
终端/命令行:  [AI Prompt ▼]
IDE/编辑器:   [AI Prompt ▼]
浏览器:       [通用润色 ▼]
其他应用:     [不润色 ▼]
```
